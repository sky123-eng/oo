# 项目启动指南

## 问题排查

如果遇到"无法打开网址"的问题，请按以下步骤操作：

## 步骤 1：安装依赖

### 方法 A：使用 PowerShell（推荐）

如果遇到执行策略错误，请先运行：

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

然后安装依赖：

```powershell
npm install
```

### 方法 B：使用 CMD

打开 CMD（命令提示符），进入项目目录，然后运行：

```cmd
npm install
```

### 方法 C：使用 Git Bash

如果安装了 Git，可以使用 Git Bash：

```bash
npm install
```

## 步骤 2：启动开发服务器

安装完成后，运行：

```bash
npm run dev
```

或者：

```powershell
npm run dev
```

## 步骤 3：访问应用

启动成功后，终端会显示类似以下信息：

```
  VITE v5.0.8  ready in 500 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

在浏览器中打开显示的地址（通常是 `http://localhost:5173`）

## 常见问题

### 1. 端口被占用

如果 5173 端口被占用，Vite 会自动尝试其他端口（如 5174、5175 等），请查看终端输出的实际地址。

### 2. 依赖安装失败

如果 `npm install` 失败，可以尝试：

```bash
# 清除缓存
npm cache clean --force

# 删除 node_modules 和 package-lock.json 后重新安装
rm -rf node_modules package-lock.json
npm install
```

### 3. 配置文件问题

确保以下文件存在且内容正确：
- ✅ `tailwind.config.js` - TailwindCSS 配置
- ✅ `postcss.config.js` - PostCSS 配置
- ✅ `vite.config.js` - Vite 配置

### 4. 浏览器无法访问

- 检查防火墙设置
- 尝试使用 `127.0.0.1:5173` 代替 `localhost:5173`
- 检查是否有代理设置影响

## 验证项目结构

确保项目包含以下目录和文件：

```
├── node_modules/          # 依赖包（安装后生成）
├── src/
│   ├── components/
│   ├── hooks/
│   ├── utils/
│   ├── styles/
│   ├── mock/
│   ├── App.jsx
│   └── main.jsx
├── index.html
├── package.json
├── vite.config.js
├── tailwind.config.js
└── postcss.config.js
```

## 如果仍然无法启动

1. 检查 Node.js 版本（需要 Node.js 16+）：
   ```bash
   node --version
   ```

2. 检查 npm 版本：
   ```bash
   npm --version
   ```

3. 查看详细错误信息，终端会显示具体的错误原因
